# Montex POS

نظام كاشير احترافي مبني باستخدام Flutter ويعمل بأسلوب `Offline First` مع قاعدة بيانات محلية `Drift/SQLite` وربط مباشر مع نظام ERP عبر API.

المشروع مصمم ليخدم نقاط البيع داخل الفروع، مع دعم البيع المحلي، الطباعة، إدارة العملاء والمنتجات، ثم مزامنة البيانات مع السيرفر عند الحاجة.

## نظرة عامة

- حفظ المبيعات محلياً داخل الجهاز حتى بدون إنترنت.
- مزامنة المنتجات والعملاء والفروع وطرق الدفع من السيرفر.
- رفع الفواتير المحلية إلى السيرفر من صفحة إعدادات المزامنة.
- دعم الطباعة الحرارية والفواتير بتنسيقات متعددة.
- دعم الفروع وطرق الدفع المرتبطة بكل فرع.
- وجود لوحة تحكم داخلية لإعدادات النظام والمبيعات والطابعات والتقارير.

## طريقة عمل النظام

1. يسجل المستخدم الدخول على النظام عبر API.
2. يتم جلب البيانات الأساسية من السيرفر مثل المنتجات والعملاء والفروع.
3. يتم تخزين هذه البيانات محلياً داخل `SQLite` عبر `Drift`.
4. عند تنفيذ عملية بيع، تحفظ الفاتورة محلياً أولاً.
5. عند تنفيذ مزامنة الفواتير، يتم رفع المبيعات التي حالتها `PENDING` أو `FAILED` إلى السيرفر.
6. بعد نجاح الرفع يتم تحديث حالة الفاتورة محلياً إلى `SYNCED` وربطها بمعرفها على السيرفر.

## شرح الوحدات الرئيسية

### `app/`

- يحتوي على التوجيه العام للتطبيق `router`.
- يحتوي على الثيم والألوان والمسافات والخطوط المشتركة.

### `core/`

- `constants`: إعدادات عامة مثل عناوين الـ API والمسارات.
- `database`: قاعدة البيانات المحلية والجداول وموفري الاتصال.
- `network`: إعداد `Dio` والاتصال بالشبكة.
- `printing`: أدوات وخطوط الطباعة والتعامل مع PDF.
- `settings`: نماذج إعدادات عامة مثل الفرع وخصائص نقاط البيع.
- `storage`: التخزين الآمن للتوكنات وبيانات الاتصال.
- `ui`: حوارات ورسائل النظام المشتركة.

### `features/auth`

- تسجيل الدخول.
- استعادة الجلسة.
- جلب بيانات المستخدم من السيرفر.
- تحديث كلمة المرور.

### `features/products`

- جلب المنتجات من السيرفر.
- مزامنة التصنيفات والعلامات التجارية.
- حفظ المنتجات محلياً.
- ربط الفروع وطرق الدفع أثناء المزامنة.

### `features/contacts`

- جلب العملاء من السيرفر.
- تحويل بيانات العملاء إلى قاعدة البيانات المحلية.

### `features/pos`

- شاشة الكاشير الرئيسية.
- السلة والدفع وتأكيد البيع.
- حفظ الفواتير محلياً.
- رفع المبيعات المعلقة إلى السيرفر.
- الطباعة والمعاينة.
- المرتجعات والمبيعات المعلقة والتسويات.

### `features/control_panel`

- لوحة تحكم للنظام.
- إدارة الطابعات.
- إدارة شاشات الإعدادات.
- التقارير.
- إدارة الحركة النقدية.
- شاشة مزامنة المنتجات والعملاء والفواتير.

- التصنيفات
- العلامات التجارية
- العملاء
- الفواتير
- عناصر الفاتورة
- دفعات الفاتورة

- بيانات المزامنة والحالة

## إعدادات API الحالية

الإعدادات الحالية موجودة في:

- [lib/core/constants/app_config.dart](lib/core/constants/app_config.dart)

القيم الأساسية حالياً:

- `defaultBaseUrl = https://erp.montexsoft.com`
- `salesUploadBaseUrl = https://test.montexsoft.com`

هذا يعني أن بعض عمليات الجلب تعمل على السيرفر الأساسي، بينما رفع المبيعات مضبوط حالياً على سيرفر رفع منفصل.

## تفاصيل المزامنة الحالية

### مزامنة المنتجات

- يتم جلب المنتجات من السيرفر.
- يتم تحديث التصنيفات والعلامات التجارية محلياً.
- يتم جلب الفروع وطرق الدفع المرتبطة بها.
- يتم حفظ آخر وقت مزامنة داخل الإعدادات المحلية.

### مزامنة العملاء

- يتم جلب العملاء من السيرفر.
- يتم حفظهم في جدول العملاء المحلي.
- يتم تحديث وقت آخر مزامنة.

### رفع الفواتير

- يتم رفع الفواتير المحلية غير المرفوعة بعد.
- يتم الاعتماد على `POST /connector/api/sell`.
- يتم رفع الفاتورة مع المنتجات ودفعاتها.
- بعد نجاح الرفع يتم حفظ `serverSaleId` محلياً.
- الفواتير التي سبق رفعها لا يعاد إنشاؤها مرة أخرى لتجنب التكرار.

## الفروع وطرق الدفع

الفروع وطرق الدفع لا تأتي من قائمة ثابتة فقط، بل يتم الآن دعم جلبها من السيرفر عبر:

- `GET /connector/api/business-location`

ويتم حفظ:

- الفروع في الإعدادات المحلية.
- طرق الدفع الخاصة بكل فرع في الإعدادات المحلية.
- الفرع المختار وبياناته لاستخدامها أثناء البيع ورفع الفواتير.
